本工程模板是针对大疆官方A板（STM32F427IIH6）进行编写的，建议使用者，根据自己所用芯片以及所需外设，使用CubeMX重新生成工程，若使用到本工程的模块层可直接复制对应.c与.h文件至自己新建工程中使用。

二、任务通知
任务通知与任务延时理解：
任务通知：我们所使用的是将任务通知模拟二值信号量，通过判断二值量信号来确定是否进入任务主体，若信号量为1则进入，反之则堵塞。任务通知发送放在对应数据接收的中断中，任务通知获取放在所需执行的任务中。
任务延时：我们通过记录任务第一次执行的时间，此后以此时间为基准，延时规定时间再次执行该任务。任务主循环执行完毕后，任务处于阻塞状态。
注：一个任务处于阻塞状态，其他就绪状态的任务可以被执行。

任务通知与任务延时使用情况：
任务通知：用于不需要的定频率执行的任务，而是在数据刷新后才执行的任务。比如遥控器任务，其主要目的是对串口接收到的遥控器数据进行进一步处理，而只有当数据刷新后，该任务执行才有意义，
故遥控器任务使用任务通知的方式执行。并不是所有的数据接收任务都需要使用任务通知的方式去执行，例如裁判系统任务，其主要是进行画UI及机器人之间的数据交互，与是否接收到裁判系统数据关系不大，
因此我们可以使用任务延时执行。
任务延时：用于定频率执行的任务，比如云台任务，其主要是对云台电机进行PID控制，而PID运算的效果与运算周期有关，所以定频率执行可以保证控制效果。
注：任务通知与任务延时不要同时使用！！！！！！！！！！！

遥控器任务：任务通知
裁判系统任务：任务延时
视觉任务：任务通知（可根据实际情况进行修改）
注：不适用任务通知时，不仅需要将接收任务通知的语句注视点，还需要将发送任务通知的语句注释！
